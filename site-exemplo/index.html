Aprendendo Git e Github

Este é um exemplo de página HTML para o curso de Git e Github.

O que é Git e Github?

O Git e Github são dois sistemas de controle de versionamento, eles são capazes de registrar todo o historico de alterações feitas em um código.

Qual a diferença entre eles?

O Git usamos localmente para ter todo o controle de versionamento dentro de nossas máquinas,tendo rastreado cada coisa que foi alterada, enquanto o Github é o lugar onde podemos alocar esses nossos códigos e colaborar com outras pessoas. No Github também é possível fazer alterações no código, mas o principal é que ele serve para hospedar os seus códigos na nuvem como se fosse um Google Drive e o Git você terá a liberdade de alterar tudo que for necessário, testar e ter toda essa rastreabilidade das alterações para hospedar no repositório remoto.

Se deseja aprender ainda mais acesse o curso do Gustavo Guanabara, clicando no link a seguir: Curso de Git e Github

Alguns comandos básicos do Git:

  • git init - Inicializa um repositório Git.
  • git add . - Adiciona todos os arquivos ao staging area.
  • git commit -m "mensagem" - Comita as alterações com uma mensagem.
  • git push - Envia as alterações para o repositório remoto.
  • git pull - Puxa as alterações do repositório remoto.
